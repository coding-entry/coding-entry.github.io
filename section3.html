Раздел 3: Условные операции и операторы, структура кода.

Из прошлого раздела вы помните, что существует булевый тип данных. Именно они будут лежать в основе этого раздела.

Условные операции - это методы, которые возвращают "истина"(True) или "ложь"(False) в зависимости от выполнения операций сравнений и логических операций.

Из самых частых операций сравнения имеются: "==" (не путать с "="! здесь идет сравнение, а не присваивание), "!=" (не равно), ">", "<", ">=" (больше либо равно), "<=".

Также часто используется in (проверяет, содержится ли элемент, стоящий впереди, в наборе данных, стоящем после).

a = 1==1

b = 2 > 4

Здесь "a" будет равно True, так как 1 равно 1, и "b" будет равно False, так как 2 не больше 4.

Операции сравнения можно объединять, используя известные вам логические операции "and"(и) и "or"(или). Также можно использовать инверсию, с помощью "not" перед условной операцией.

В Python разрешается заключать условные и другие операции в скобки не только для визуальной читаемости, но и для установки порядка чтения. Например "(1 == 1 and 2 == 2) or 3 == 3" не то же самое, что и "1 == 1 and (2 == 2 or 3 == 3)". Пробелы с двух сторон от and и or обязательны!

Как вы могли заметить, часто среди примеров используются пробелы между числами, переменными и т.д. В Python и во многих других языках поддерживается разделение знаков для удобства. Например: a = 1 вместо a=1 ; b = a - 2 вместо b=a-2. С операциями сравнения это тоже работает.

Условные операции редко присваиваются переменным. Взамен, они необходимы для использования условных операторов.

Условные операторы - комманды, с помощью определяется, какая часть кода будет выполнена, в зависимости от результата условных операций.

Самым распространённым и самым простым является if и else.

Оператор if принимает после себя некоторое булевое значение. Оно может быть получено из переменной или напрямую с помощью условных операций. Если булевое значение равно "True", то следующий блок кода будет выполнен.

Оператор else не может использоваться без прежде стоящего if, так как оно выполняет противоположную ему функцию - блок кода будет выполнен, только если булевое значение в операторе if будет равно "False".

Для сокращения также существует гибрид - оператор elif. Как и else, он идет после if, но также включает в себя проверку булевого значения, как в обычном if. В таком случае, блок кода, указанный для elif, будет выполнен в том случае, когда условие первого if будет отрицательным (False), но собственное условие будет положительным (True).

Несколько раз были упомянуты блоки кода. Это важная часть программирования. Чтобы объяснять компьютеру и человеку, какая часть кода относится к чему, и при каких условиях выполняется, участки кода выносятся в качестве блоков. Все строчки в этом блоке должны начинатся с определенным отступом (по умолчанию 4 символа пробела, нажатием кнопки Tab можно быстро сделать отступ в редакторе кода), причем при наличии блоков в блоках (не редкость) эти отступы складываются.

В других языках программирования блоки могут быть реализованы с помощью фигурных скобок. Их границы означают границы блока.

На примере оператора if разберем блочность:

if 1 == 2:

print("Да")

else:

print("Нет")

Каждый блок кода должен начинаться в конце какого либо оператора, который подразумевает использование блоков. Это обозначается ":" в конце оператора. После этого, все строчки, имеющие отступ, будут относится к блоку. В этом случае, если бы 1 было равно 2, то выполнилась бы одна строчка "print("Да")". Так как 1 не равно 2, блок под if пропускается компьютером, и запускается блок под else.

Примеры условных операторов:

if age > 18:

adult_count += 1

else:

children_count += 1

if alive:

people_count += 1

#...где alive - это булевая переменная.
